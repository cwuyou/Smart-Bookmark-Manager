# 需求文档

## 介绍

修复书签分组卡片拖拽功能的问题。当前系统使用 CSS Grid 布局显示分组卡片（每行3个），但拖拽功能存在缺陷：第一行的卡片可以正常拖拽重新排序，但从第二行开始的卡片在拖拽时无法放置到本行，只能放置到第一行。这个问题影响了用户体验，需要修复以实现所有行的卡片都能自由拖拽重新排序。

## 需求

### 需求 1

**用户故事：** 作为用户，我希望能够在任意行内拖拽分组卡片重新排序，以便更好地组织我的书签分组。

#### 验收标准

1. WHEN 用户拖拽第一行的分组卡片 THEN 系统 SHALL 允许在第一行内任意位置放置
2. WHEN 用户拖拽第二行及以后行的分组卡片 THEN 系统 SHALL 允许在同行内任意位置放置
3. WHEN 用户拖拽任意行的分组卡片 THEN 系统 SHALL 允许跨行放置到其他行的任意位置
4. WHEN 拖拽操作完成 THEN 系统 SHALL 保持卡片的新顺序并保存到本地存储

### 需求 2

**用户故事：** 作为用户，我希望拖拽操作有清晰的视觉反馈，以便了解当前的拖拽状态和可放置位置。

#### 验收标准

1. WHEN 用户开始拖拽分组卡片 THEN 系统 SHALL 显示拖拽状态的视觉反馈
2. WHEN 拖拽的卡片悬停在有效放置区域 THEN 系统 SHALL 显示可放置的视觉提示
3. WHEN 拖拽操作进行中 THEN 被拖拽的卡片 SHALL 保持适当的透明度或样式变化
4. WHEN 拖拽操作结束 THEN 系统 SHALL 恢复正常的视觉状态

### 需求 3

**用户故事：** 作为用户，我希望拖拽功能在不同屏幕尺寸下都能正常工作，以便在各种设备上使用。

#### 验收标准

1. WHEN 在桌面端（lg断点）显示3列布局时 THEN 拖拽功能 SHALL 在所有行正常工作
2. WHEN 在平板端（md断点）显示2列布局时 THEN 拖拽功能 SHALL 在所有行正常工作  
3. WHEN 在移动端显示1列布局时 THEN 拖拽功能 SHALL 正常工作
4. IF 屏幕尺寸改变 THEN 拖拽功能 SHALL 继续正常工作而不需要页面刷新
